# Project Docs

This folder is the working context for the hackathon build.

## Files

- `product-requirements.md`: distilled requirements from the PDF
- `execution-plan.md`: 7-hour delivery plan and build order
- `architecture-and-data-model.md`: app structure, entities, routes, and Supabase schema direction
- `credentials-and-deployment.md`: credentials checklist, env vars, and repo/deploy steps
- `features-and-e2e-testing.md`: complete feature overview and manual end-to-end testing flow
- `mvp-readiness-checklist.md`: final pre-demo and pre-deploy verification checklist
- `vercel-ui-deployment-guide.md`: step-by-step Vercel dashboard deployment guide
- `demo-script.md`: short presentation flow for judges
- `progress-tracker.md`: completed work, pending items, and next priorities

## How to use this folder

- Treat `product-requirements.md` as the source of truth for scope
- Treat `execution-plan.md` as the implementation order
- Treat `progress-tracker.md` as the running status board
- Update `credentials-and-deployment.md` when you receive the hackathon repo details
- Keep new decisions documented here so the project does not drift

## Documentation maintenance rule

- Update the relevant docs whenever code changes alter behavior, routes, setup steps, data flows, or project status
- Keep `progress-tracker.md` aligned with the actual implementation, not the original plan
- Update the root `README.md` when the public project summary changes
